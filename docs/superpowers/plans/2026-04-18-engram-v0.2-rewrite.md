# engram v0.2 全面重写实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development`(推荐)或 `superpowers:executing-plans` 来逐任务执行。每个 step 用 checkbox(`- [ ]`)跟踪。

**Goal:** 把 engram 从 v0.1(3 层:数据 / CLI / 适配器)推倒重写为 v0.2(5 层:数据 / CLI / 智能 / 接入 / 观察),落实五大支柱、一致性引擎、4 级 scope、跨仓 inbox、三类资产(Memory / Workflow / KB)、自演化 Wisdom Metrics,并双语(英文 canonical + `*.zh.md` 中文同级)发布,目标成为业界最好的开源永久记忆系统。

**Architecture:**
- **5 层架构**:Layer 1 数据 → Layer 2 CLI → Layer 3 智能 → Layer 4 接入 → Layer 5 观察
- **三类资产**:Memory(短,LLM 高频)/ Workflow(文档 + 可执行 spine,LLM 执行)/ Knowledge Base(长,人写 LLM 编译)
- **4 级 scope**:team > global > pool > local,每条带 `enforcement: mandatory | default | hint`
- **核心差异化**:一致性引擎(7 类冲突检测 + 4 层扫描 + 证据驱动置信度)+ 跨仓 inbox + 自演化 Wisdom Metrics
- **哲学锁死**:不设容量上限、永不自动删、只建议不自动 mutate、质量 > 容量

**Tech Stack:**
- 文档本身:markdown + YAML frontmatter(同 spec)
- 未来 CLI:Python + click(pyproject.toml)
- 未来 Web:FastAPI + Svelte + D3 + CodeMirror 6 + SSE
- 未来存储:SQLite(graph.db、cache)+ 纯文件系统(markdown)
- 未来 embedding:bge-reranker-v2-m3(本地)+ 可选 OpenAI/Cohere(云端)

**本 plan 的作用域**:只产出文档(SPEC / DESIGN / TASKS / README / METHODOLOGY / CONTRIBUTING 的 v0.2 双语版 + 术语表 + plan 归档)。不碰 CLI 代码。CLI / Web 的代码实现是 v0.2 发布后的下一阶段,由 TASKS v0.2 驱动。

### B.1 资产尺寸:无硬上限,函数定义 + 3 自适应信号

**删除**所有固定行数建议("典型 1-20 行"、"soft ceiling 300"、"X-Y 行"等)。Memory / Workflow / KB **按函数**区分:

| 资产 | 定义 | 必备结构 |
|------|------|----------|
| Memory | 原子断言(能独立 supersede 的事实 / 规则 / 偏好 / 指针) | 一个 `.md` + frontmatter |
| Workflow | 可执行过程(有 spine 能跑) | `workflow.md` + `spine.*` + `fixtures/` + `metrics.yaml` + `rev/` |
| Knowledge Base | 领域参考(独立导读 + 多章节) | `README.md` + 章节 + `assets/` + `_compiled.md` |

**3 个自适应信号代替固定数字:**

1. **Relevance Gate 动态预算分配**(DESIGN §5.1 扩展)
   - 按 utilization rate 自适应调整类型预算
   - 冷启动默认:feedback > user > project > reference > workflow_ptr > kb_compiled
   - 几周数据后按 Wisdom Metrics 动态重分配

2. **百分位长度信号**(DESIGN §5.2 Consistency Engine,**M4 落地**)
   - 项目内按资产类型分别统计长度分布
   - ≥95 百分位触发 `engram review` attention 项(不阻塞、不警告、不拒绝)
   - 阈值随项目分布动态变,**不是全局数字**

3. **Evolve Engine split/promote/demote proposal**(DESIGN §5.3 扩展,**M5 落地**)
   - split:一条 memory 内 2+ 不相交语义簇 + 部分引用 / 部分忽略
   - promote to KB:3+ memory 共现 >60% + 语义相关 + 总量大
   - demote workflow to memory:spine 90 天无执行,只 doc 被读
   - promote memory to workflow:memory 含 "do X then Y then Z" + 多次用于同类任务
   - 全是 proposal,人 / LLM review 决定,**永不自动执行**

**硬约束只有一个:context budget**(LLM token 上限) —— Relevance Gate 动态处理,与磁盘资产尺寸无关。长 memory 紧张时被降权,不紧张时完整加载,**绝不截断**。

### B.2 Relevance Gate 采用 MemPalace Hybrid v2 作为 baseline

**DESIGN §5.1 重写**。移植 MemPalace 已证实的 98.4% R@5 算法,加 engram 的 scope/enforcement 加权差异化。

**核心算法:**

```python
# Stage 1: 语义 retrieval
candidates = vector_search(query, top_k=50)

# Stage 2: BM25 关键词重排
fused_dist = dist * (1.0 - 0.30 * keyword_overlap)

# Stage 3: 时间邻近 boost(如 query 含"N weeks ago")
if has_temporal_anchor:
    temporal_boost = max(0.0, 0.40 * (1.0 - days_diff / window_days))
    fused_dist *= (1.0 - temporal_boost)

# Stage 4: Two-pass for assistant-reference queries
if query_references_assistant:
    top_sessions = vector_search(user_turns_only, query, top_k=5)
    full_text_coll = build_coll(top_sessions, include_assistant=True)
    candidates = vector_search(full_text_coll, query, top_k=5)

# Stage 5: engram 独有 —— scope/enforcement 加权
# (project=1.5, user=1.2, team=1.0, org=1.5-mandatory-weighted;pool 按 subscribed_at)
# mandatory 强制进 context 绕过分数
```

- 停用词表 32 词(直接复用 MemPalace 的)
- 3+ 字符 token 作为 keyword
- Target:LongMemEval Relevance Gate 单独测 ≥95% R@5(对标 MemPalace 96.6%,非最终指标,仅 baseline)
- **engram 的突破不在 retrieval 精度,在 scope/enforcement 的语义正确性**

**影响:** T2.4(DESIGN §5.1)按此重写。

### B.5 Temporal validity 字段(SPEC §4 frontmatter 新增可选)

- `valid_from: ISO 8601 date`(可选):事实成立的起始日期
- `valid_to: ISO 8601 date`(可选):事实成立的结束日期
- 与现有 `expires` 语义互补:
  - `expires`:一个截止日,此后可能过期
  - `valid_from` / `valid_to`:一段区间,这段时间内为真
- graph.db 支持时间过滤查询("2026-03-01 时以下哪些事实为真?")
- Consistency Engine 的 `time-expired` 类检测看 `expires` 和 `valid_to` 两个字段

### B.6 Autolearn Engine:Darwin 式 ratchet + 双评 + 独立评审 + phase gate

**DESIGN §5.3 扩展**,4 条 Darwin 模式全采纳:

- **G1 Ratchet(git-native)**:每轮 autolearn = 对 `workflows/<name>/rev/rN/` 一个 commit。metric 降 → 自动 revert(current symlink 回前一版)。metric 升 → keep。`evolution.tsv` append-only
- **G2 双维度评分 60+40**:
  - 静态 60:SPEC validate + fixtures 齐 + spine 可解析 + metrics.yaml 合规 + 无 secret leak
  - 动态 40:所有 fixtures 通过 + metric delta > 0
  - 总分 100,pass threshold 默认 70,可配
- **G3 独立评估 agent**:proposer subagent 和 judge subagent 用**不同** context,防自我打分偏见
- **G4 Phase gate**:连续 K=5 轮后强制 pause,diff summary 写 `engram review`,等 human confirm 继续。夜间不让跑 500 轮没人管

**影响:** T2.6(DESIGN §5.3)按此重写。
