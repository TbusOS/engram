acme-checkout-architecture

1. 概述

acme-checkout-service 负责处理 Acme 平台的所有购买流程：购物车管理、支付处理、库存预留、收据发送和订单确认。峰值时每小时处理约 4,200 笔交易。

该服务是一个 Python FastAPI 应用，以 3 个副本的形式部署在 Nginx 入口后。服务本身不存储持久状态 — 所有状态存放在 PostgreSQL（订单）、Redis（购物车会话）和支付方账本中。

2. 核心服务

结账服务协调 4 个下游系统。每次调用都套用了熔断器，配置为 2 秒超时和 3 次重试。

服务 协议 SLA 降级策略
payment-provider HTTPS REST 99.95% 入队重试
inventory-svc gRPC 99.9% 软预留
notifications-svc AMQP（异步） 99.5% 记录日志并在下一小时重试
fraud-svc HTTPS REST 99.9% 转人工审核队列

3. 数据流

一次结账请求经历 5 个阶段：购物车校验 → 欺诈检测 → 支付授权 → 库存预留 → 确认通知。每个阶段在调用下一个服务之前，都会向 orders 表写入一条结构化事件。

说明

从用户角度看，第 3 阶段和第 4 阶段是原子操作，但在系统层面并非如此 — 支付可能成功而库存预留失败。Saga 模式负责处理回滚；恢复步骤详见第 11 章运维手册。
